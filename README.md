    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-next-generation-sequence-analysis-toolkit-nest--a-standardized-bioinformatics-framework-for-analyzing-snps-in-next-generation-sequencing-data" class="anchor" aria-hidden="true" href="#next-generation-sequence-analysis-toolkit-nest--a-standardized-bioinformatics-framework-for-analyzing-snps-in-next-generation-sequencing-data"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Next-generation Sequence-analysis Toolkit (NeST) : A standardized bioinformatics framework for analyzing SNPs in next-generation sequencing data</h1>
<p>Advancements in next-generation sequencing have led to the development of numerous bioinformatics tools and pipelines. Current tools for variant calling offer high-quality solutions; however, many tools are tailored for model organisms. Here, we present NeST, a consensus-based variant calling tool with a plug-and-play framework for use with any organism with minimal user input. NeST consists of four modules, integrating open-source bioinformatics tools and a custom VCF parser, to generate high-quality consensus variant calls. NeST was validated using targeted-amplicon deep sequencing data from 245 Plasmodium falciparum isolates to identify single-nucleotide polymorphisms conferring drug resistance. NeST offers a light-weight pipeline for variant calling with standardized outputs and minimal computational demands for easy deployment for use with various organisms and applications. The following document outlines details of installation, results from MaRS dataset and usage of individual modules for analysis.</p>

<h2><a id="user-content-overview-of-the-nest-framework" class="anchor" aria-hidden="true" href="#overview-of-the-nest-framework"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Overview of the NeST framework:</h2>
<p>NeST is a python based modular framework for consensus based variant calling. The overall analysis framework is broken down into four major blocks.</p>
<ol>
<li>PrepInputs</li>
<li>VarCallEngine</li>
<li>VCFToolkit</li>
<li>Summarize</li>
</ol>
<p><a target="_blank" rel="noopener noreferrer" href="/CDCgov/NeST/blob/master/images/Kookaburra.png"><img src="/CDCgov/NeST/raw/master/images/Kookaburra.png" alt="NeST framework overview" style="max-width:100%;"></a></p>
<p>The figure outlines the four key blocks of NeST and the steps performed by each step. VarCallEngine and VCFToolkit are spawned in parallel for each sample that is being analyzed in the study. By default, 4 parallel threads are spawned, to account for minimum available computational resource. This can be altered as per availability of resources.</p>

<p><a id="user-content-first"></a></p>
<h2><a id="user-content-your-first-analysis" class="anchor" aria-hidden="true" href="#your-first-analysis"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Your first analysis</h2>
<p>NeST was conceptualized to identify mutations that confer anti-malarial drug resistance in <em>P.falciparum</em> (Talundzic et al., 2018). It was also applied for the detection of antibiotic drug resistance in <em>M.tubercolosis</em> (Colman et al., 2015). To make it easier to recreate these studies, we have included script to execute NeST on these two datasets.</p>
<ol>
<li>
<p>MaRS on NeST:</p>
<p>To re-create the MaRS study (Talundzic et al., 2018), run the following command from the NeST directory.</p>
<pre><code>./runPF.sh
</code></pre>
<p>This downloads 10 sample fastq files from the [MaRS bioproject]((<a href="https://www.ncbi.nlm.nih.gov/bioproject/?term=PRJNA428490" rel="nofollow">https://www.ncbi.nlm.nih.gov/bioproject/?term=PRJNA428490</a>) and executes NeST on the samples. The results are stored under <code>local/MaRS_test</code> folder within the NeST directory. The 10 target amplicon sequencing datasets, serve as a perfect test case for NeST and the analysis should take about 10 minutes on any laptop*.</p>
<p><em>*Depending on internet speed</em></p>
</li>
<li>
<p>Insilico datasets for NeST valiadation:</p>
<p>To compare the accuracy of the three variant callers included with NeST, we generated 108 insilico datasets from 7 genes assosicated with anti-malarial drug resistance in <em>P. falciparum</em>. The detailed sample list and simulated error rates and mutation rates can be found in the <code>fq/Insilico/Plasmodium_insilico_study.tsv</code>. The FASTQ files for this analysis can be generated by following the steps shown here</p>
<pre><code>cd fq/Insilico
python3 simulator.py
</code></pre>
<p>To run NeST on the dataset use the following command, the outputs will be stored in the <code>local/Insilico</code> folder within the NeST folder</p>
<pre><code>python3 nest.py -i fq/Insilico/ -a ref/pfalciparum/adapters.fa -r ref/pfalciparum/mdr.fa -o local/Insilico -b ref/pfalciparum/mdr.bed -m bowtie2 --varofint ref/pfalciparum/Reportable_SNPs.csv
</code></pre>
</li>
<li>
<p>Malaria Reasistance Surveillance Project:</p>
<p>The study introduced a new library preparation method to indetify and track mutations that are associated with anti-malarial drug resistance. The samples were seuqenced using Illumina MiSeq to generate 243 paired-end targeted amplicon deep sequencing datasets. These variant calling was performed on these samples using NeST and the results were compared against the variant calls from Sanger sequencing of the same samples, as well Geneious, a commercial NGS data analysis software. To call variants on these 243 samples using NeST, run the following command</p>
<pre><code>python3 nest.py -i fq/MaRS/Accession_list.txt -a ref/pfalciparum/adapters.fa -r ref/pfalciparum/mdr.fa -o local/MaRS_sanger -b ref/pfalciparum/mdr.bed -m bowtie2 --varofint ref/pfalciparum/Reportable_SNPs.csv
</code></pre>
</li>
<li>
<p>Detecting mutations conferring drug resistance in <em>M.tuberculosis</em> clinical samples from Colman et al., 2015:</p>
<p>To re-create the analysis from Colman et al., 2015, run the following command from the NeST directory.</p>
<pre><code>./runTB.sh
</code></pre>
<p>This will download 57 paired fastq files from the bioproject <a href="https://www.ncbi.nlm.nih.gov/bioproject/PRJNA271805" rel="nofollow">PRJNA271805</a>, merges the different runs and executes NeST on the clinical samples from the paper. The results are stored under <code>local/ColmanEtAl</code> folder within the NeST directory. This analysis takes significantly longer than the MaRS study and requires around 80GB of disk space, due to the large amount of data that is download. Make sure you have a good internet connection and adequate amount of coffee before starting this study.</p>
</li>
<li>
<p>Executing NeST on a HPC framework:</p>
<p>An early implementation of HPC enabled NeST is available with the <a href="https://github.com/shashidhar22/NeST/tree/v2.1.0#hpc">NeSTv2.1.0 branch</a>. The script <code>nest_submitter.py</code> generates and submits a job in a Moab scheduler environment. It is still a work in progress and users need to manipulate the nest_submitter.py to reflect the HPC enivronment specifications. The command listed below will spawn 101 jobs for all the 10105 samples from <a href="https://www.nejm.org/doi/full/10.1056/NEJMoa1800474" rel="nofollow">(The CRyPTIC Consortium and the 100, 2018)</a> paper, using 1 node per job, with 15 threads and 16gb of memory per threads. Since this is still a work in progress, please use make necessary changes to the submitter script or create an issue on the GitHub branch with your HPC framework details and we will get back to you with the changes needed to execute NeST on your HPC framework.</p>
<p>The average wall time for 101 jobs was 3.84 hours. The ability to split samples and schedule parallel jobs, drastically reduces the turn around time for any analysis. The results for each of the 101 jobs can be downloaded from <a href="http://vannberg.biology.gatech.edu/data/NEJM_results.tar.gz" rel="nofollow">http://vannberg.biology.gatech.edu/data/NEJM_results.tar.gz</a></p>

<p>Executing your own analysis using NeST:</p>
<p>NeST can be executed on your own dataset using the following command:</p>
<pre><code>python3 nest.py -i &lt;path to input directory/ sra accession list&gt; -a &lt;adapter fasta file&gt; -r &lt; reference fasta file&gt; -b &lt; reference bed file&gt; -o &lt;output directory path&gt; --varofint &lt;CSV file with variants of interest&gt;
